# 碳中和数字碳资产管理系统

## 1. 项目介绍

本项目是一个基于微服务架构的数字碳资产管理平台，旨在为企业或组织提供一个全面、高效的碳资产数字化解决方案。系统支持对碳资产的发行、追踪、交易、审批和审计等全生命周期管理，帮助用户实现碳资产的价值最大化，并助力企业达成碳中和目标。

系统采用前后端分离的设计思想，前端负责用户交互与数据展示，后端提供稳定、高效的业务逻辑和数据服务。通过模块化的设计，系统具备良好的可扩展性和可维护性。

## 2. 前后端代码结构

### 2.1 后端代码结构 (Java Microservices)

后端采用 Spring Boot 和 Spring Cloud 构建微服务体系，代码结构清晰，职责分明。

```
carbon-system/
├── carbon-common         # 公共模块：存放通用工具类、实体、注解、统一返回结果等
├── service-consumer      # 服务消费者模块：面向前端，聚合后端服务，通过 Feign 调用服务提供者
└── service-provider      # 服务提供者模块：核心业务实现，提供原子化的服务接口，与数据库交互
```

*   **`carbon-common`**: 项目的基石，包含了被其他所有模块共享的核心定义，如 `Result` 统一响应体、`@RequireLogin` 等自定义注解、以及通用的数据模型。
*   **`service-provider`**: 核心业务逻辑的实现中心。它直接与数据库（MySQL）交互，通过 MyBatis-Plus 实现数据的持久化操作，并向上提供原子化的 RESTful API 接口。我们之前调试的 `User`、`SysOperationLog` 等实体及其 Service 和 Controller 都位于此模块。
*   **`service-consumer`**: 作为前端与后端微服务之间的网关和桥梁。它接收来自前端的 HTTP 请求，通过 Feign 客户端以 RPC 的方式调用 `service-provider` 提供的接口来完成业务逻辑，最后将结果返回给前端。

### 2.2 前端代码结构 (Vue.js)

前端是一个基于 Vue.js 框架开发的单页面应用（SPA），具有现代化的工程结构。

```
frontend/
├── public/               # 静态资源，如 index.html, favicon.ico
├── src/                  # 核心源代码目录
│   ├── api/              # API 请求封装，如 user.js, log.js
│   ├── assets/           # 静态资源，如 CSS, 图片
│   ├── components/       # 可复用的全局 Vue 组件
│   ├── router/           # 路由配置 (Vue Router)
│   ├── store/            # 状态管理 (Pinia)，如 user.js
│   ├── utils/            # 工具函数，如 request.js (Axios 封装)
│   ├── views/            # 页面级 Vue 组件，如 system/user.vue, system/operation.vue
│   ├── App.vue           # 根组件
│   └── main.js           # 应用入口文件
├── .env.development      # 开发环境变量配置
├── package.json          # 项目依赖与脚本配置
└── vite.config.js        # Vite 构建配置
```

## 3. 项目各模块说明

系统主要包含以下几个核心功能模块：

*   **系统管理 (`system`)**:
    *   **用户管理**: 维护系统用户信息，包括用户的增删改查、状态管理等。
    *   **角色管理**: 配置用户的角色，定义不同角色的权限集合。
    *   **菜单管理**: 动态配置系统导航菜单，并与权限标识挂钩。
    *   **操作日志**: 记录系统关键操作，便于审计和问题追溯（我们刚刚修复的功能）。
    *   **字典管理**: 维护系统中使用的各类静态数据字典，如操作类型、状态等。
    *   **参数配置**: 提供系统级的可配置参数。

*   **审批流程管理 (`approval`)**:
    *   **流程定义**: 可视化或通过配置定义不同业务（如资产发行、大额交易）的审批流程。
    *   **审批记录**: 查看和管理所有提交的审批申请实例。
    *   **审批历史**: 追踪每个审批实例的详细流转过程和处理意见。

*   **核心业务模块 (推断)**:
    *   **数字碳资产管理**: 对碳资产进行登记、发行、冻结、注销等操作。
    *   **碳资产交易**: 提供用户间的碳资产交易撮合功能。
    *   **数据看板/仪表盘**: 以图表形式展示碳资产总量、交易趋势、碳排放数据等关键指标。

### 4.1 后端技术

| 技术          | 说明                             |
|---------------|----------------------------------|
| Spring Boot   | 核心开发框架，简化应用构建与开发 |
| Spring Cloud  | 微服务治理框架                   |

| Vite          | 新一代前端构建工具                 |
| Vue Router    | Vue.js 官方路由管理器              |
| Pinia         | Vue.js 新一代状态管理库            |
| Axios         | 基于 Promise 的 HTTP 请求库        |
| Element Plus  | 基于 Vue 3 的桌面端组件库          |

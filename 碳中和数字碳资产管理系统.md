# 碳中和数字碳资产管理系统

## 1. 项目介绍

本项目是一款专为响应国家“双碳”战略目标而设计的企业级数字碳资产管理系统。系统采用先进的微服务架构开发，旨在帮助企业实现碳资产的全生命周期数字化管理。

核心功能覆盖了从碳配额（Quota）的下发与履约、自愿减排量（CCER/Credit）的项目开发与签发，到碳资产的内部调拨与市场化交易撮合。通过内置的自动化审批流程引擎和实时交易撮合机制，本系统能够显著提升企业碳资产的盘查效率与流转透明度，助力企业高效达成绿色可持续发展目标，实现碳资产的保值增值。

## 2. 整体代码结构

### 2.1 后端代码结构 (Java / Spring Cloud)

后端采用 Maven 多模块（Multi-Module）架构，各模块职责划分清晰，便于扩展与维护：

* **`carbon-system` (根项目)**
    * 聚合父工程，统一管理所有子模块的构建版本与依赖（Dependency Management）。
* **`carbon-common` (公共模块)**
    * 提供系统通用的基础设施，包括：
        * `util`: 通用工具类（JwtUtils, Result 等）。
        * `security`: 安全上下文与登录用户信息封装（UserContextHolder）。
        * `annotation`: 自定义注解（@RequireRole, @OperationLog 等）。
        * `model`: 统一 API 响应结构（Result）。
* **`service-provider` (核心系统服务)**
    * 系统的基础设施与权限中心，负责：
        * 用户、角色、菜单的 RBAC 权限管理。
        * 系统配置与数据字典管理。
        * 审批流引擎（SysApproval*）的核心逻辑。
        * 企业基础信息管理。
* **`service-consumer` (网关与聚合服务)**
    * 作为前端的统一入口，负责：
        * 聚合各微服务的数据，提供给前端展示（如 Dashboard 数据）。
        * 集成 Feign 客户端，实现服务间的远程调用与降级处理。
* **`service-assets` (碳资产管理服务)**
    * 核心业务模块，负责：
        * **碳配额 (Quota)**：管理企业配额的总量、发放与履约记录。
        * **碳信用 (Credit)**：管理核证自愿减排量的签发与持有详情。
        * 资产变更的合法性校验逻辑。
* **`service-trade` (交易撮合服务)**
    * 碳市场的核心引擎，负责：
        * **挂单管理**：买单与卖单的发布（TradeOrder）。
        * **撮合引擎**：定时任务（TradeMatchScheduler）执行买卖单匹配。
        * **交易记录**：生成成交记录（TradeRecord）。
        * **资金账户**：管理企业的交易资金账户。
* **`service-development` (项目开发服务)**
    * 负责碳减排项目的全流程管理：
        * 项目立项申报与文档管理（ProjectDocument）。
        * 方法论（Methodology）库管理。
* **`service-news` (资讯服务)**
    * 负责行业新闻、政策公告的发布与查询。

### 2.2 前端代码结构 (Vue 3 / Vite)

前端项目位于 `frontend` 目录，基于现代化的 Vue 3 生态构建：

* **`src/api/`**：统一管理 HTTP 请求，按业务模块封装（如 `assets.js`, `trade.js`, `system.js`）。
* **`src/views/`**：页面视图组件，按功能模块划分：
    * `dashboard/`：首页数据看板，展示核心统计指标。
    * `assets/`：资产管理页面（配额管理 `quota.vue`、信用管理 `credit.vue`）。
    * `market/`：交易中心页面（行情 `quotes.vue`、撮合 `exchange.vue`、持仓 `account.vue`）。
    * `development/`：项目开发页面（工作台 `workbench.vue`、方法论 `methodology.vue`）。
    * `system/`：系统管理页面（用户、角色、审批流配置等）。
    * `account/`：企业账户信息设置。
* **`src/store/`**：状态管理库（Pinia/Vuex），处理用户登录状态与全局配置。
* **`src/layout/`**：通用布局组件（侧边栏、顶部导航、内容区域）。
* **`src/utils/`**：工具函数，包括 Axios 拦截器封装（`request.js`）。
* **`src/router/`**：路由配置，定义页面跳转规则与权限守卫。

## 3. 项目各模块说明

| 模块名称 | 对应服务 | 功能详述 |
| :--- | :--- | :--- |
| **RBAC 权限管理** | `service-provider` | 实现基于角色的访问控制，支持用户、角色、菜单的增删改查。通过拦截器实现接口级的权限鉴权，保障系统安全。 |
| **审批流引擎** | `service-provider` | 内置自定义审批流程，支持配置审批节点（Node）、定义审批流（Flow），并记录完整的审批历史（History），用于资产变更和项目立项的审核。 |
| **碳资产管理** | `service-assets` | 企业的“碳钱包”。实时记录企业拥有的碳配额（Quota）和碳信用（Credit）数量，提供资产的冻结、解冻及扣减接口，确保资产数据的强一致性。 |
| **交易撮合系统** | `service-trade` | 模拟二级市场交易。支持限价订单发布，后台通过定时任务进行买卖队列的匹配撮合，自动生成交易流水并更新账户资金与资产余额。 |
| **碳资产开发** | `service-development`| 协助企业开发碳资产。管理从项目立项、文档上传到方法论引用的全过程，支持项目进度的可视化追踪。 |
| **数据看板 (Dashboard)** | `service-consumer` | 聚合展示系统核心指标，包括碳资产总量分布、近期交易趋势图、待办审批事项等，为决策提供数据支持。 |

## 4. 所用技术清单

### 4.1 后端技术栈
* **开发语言**：Java 1.8+
* **核心框架**：Spring Boot 2.7.x
* **微服务架构**：Spring Cloud & Spring Cloud Alibaba (Nacos, OpenFeign)
* **持久层框架**：MyBatis-Plus
* **数据库**：MySQL 8.0
* **鉴权安全**：JWT (JSON Web Token), 自定义拦截器
* **工具库**：Lombok, Hutool, FastJSON

### 4.2 前端技术栈
* **核心框架**：Vue.js 3 (Composition API)
* **构建工具**：Vite
* **路由管理**：Vue Router 4
* **状态管理**：Pinia / Vuex
* **UI 组件库**：Element Plus (推荐使用) / Ant Design Vue
* **HTTP 客户端**：Axios
* **图表库**：ECharts (用于 Dashboard 数据可视化)

# 后续开发计划与建议

## 当前进度 (2023-XX-XX)
- **已完成**:
  - 基础微服务架构搭建 (Consumer, Provider, Gateway/Nacos配置).
  - 用户体系 (User): 注册、登录、信息获取.
  - 碳资产模块 (Carbon Credit):
    - 数据库表 `carbon_credit` 设计与初始化.
    - Provider端: Entity, Mapper, Controller.
    - Consumer端: FeignClient集成, 用户ID自动解析.
    - 前端: API定义 (`assets.js`), 页面 (`credit.vue`) 数据对接.

## 建议后续开发 roadmap

### 1. 完善碳资产管理 (Current Phase)
- **碳配额 (Quota)**: 仿照 `CarbonCredit` 的模式，实现 `CarbonQuota` 的后端与前端对接.
- **资产详情页**: 点击资产列表中的"详情"，展示具体的签发记录、有效期等详细信息.
- **资产操作**: 实现"挂牌" (改为交易中状态) 和 "注销" 的后端接口与前端交互.

### 2. 碳交易市场 (Market Module)
- **核心功能**:
  - **挂单 (Order)**: 买单/卖单的创建，关联具体的碳资产ID.
  - **行情大厅**: 展示当前活跃的挂单列表.
  - **交易记录**: 记录成交历史.
- **技术点**:
  - 需要在 Provider 端增加 `TradeOrder` 表.
  - 实现简单的撮合逻辑或点对点成交逻辑.

### 3. 碳资产开发 (Development Module)
- **核心功能**:
  - **项目立项**: 提交新的减排项目申请.
  - **文档管理**: 上传项目设计文件 (PDD) 等.
- **技术点**:
  - 文件上传接口 (可能需要 MinIO 或简单的本地文件存储).
  - 状态机管理 (项目审批流程).

### 4. 仪表盘 (Dashboard)
- **核心功能**:
  - 聚合数据展示: 总资产、总交易额、最新通知.
- **技术点**:
  - 聚合查询接口 (DashboardController), 组装各个模块的统计数据.

## 目录结构规范建议
- **API命名**: `/api/{module}/{resource}/{action}` (e.g., `/api/assets/credit/list`)
- **前端结构**: `views/{module}/{page}.vue` 对应 `api/{module}.js`
